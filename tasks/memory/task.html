Aufgabe: Erstelle ein Merkspiel mit verschiedenen Optionen. Nutze hierfür die Bilder von Dako-Mitarbeitern

  1. Erstelle einen Counter, der gut sichtbar ist und auf 0 steht
  2. Erstelle das Spielbrett
    1. Erstelle ein quadratisches Div mit schwarzem Rahmen
    2. Auf dem Feld befinden sich 16 kleine, quadratische Divs mit jeweils schwarzen Rahmen
    3. Es befinden sich jeweils 4 Divs nebeneinander in einer Reihe
    4. Ordne immer 2 zufälligen Divs ein züfälliges Bild aus dem Array "Cards" zu
    5. Die Bilder sind für den Anwender nicht sichtbar
  3. Klickt der Nutzer auf ein kleines Div, passiert folgendes:
    1. Das zugeordnete Bild wird innerhalb des geklickten Divs angezeigt
    2. Klickt der Nutzer auf ein weiteres Div, treten folgende Dinge auf:
      • Der Counter wird um eins erhöht
      • Das Bild zum zweiten Div wird angezeigt
      • Ist das Bild nicht das selbe wie vom ersten Div, werden beide Bilder nach 3 Sekunden wieder ausgeblendet
  4. Sind alle Bilder sichtbar, wird eine Meldung ausgegeben, dass der Spieler gewonnen hat und wieviele Züge er benötigt hat (Counter)
  5. Füge einen Button hinzu, der das Spiel startet. Beim Laden der Seite ist das Spielfeld also nicht spielbar.
  6. Füge 2 Radiobuttons hinzu
    1. Ein Button hat das Label "Bild - Bild" und der andere "Bild - Name"
    2. Standardmäßig ist "Bild - Bild" ausgewählt
    3. Ist "Bild - Name" ausgewählt, wird einem zufälligen kleinem Div ein Bild zugeordnet und einem anderen zufälligen Div der passende Name zum Bild
    4. Klickt der Spieler auf ein kleines Div, dass einem Namen zugeordnet ist, wird dieser im jeweiligen Div angezeigt
    5. Findet ein Spieler den passenden Namen zu einem Bild oder umgekehrt, bleibt der Name und das Bild sichtbar
    6. Erst, wenn der Spieler auf den Start-Button drückt, wird die ausgewählte Option angewendet und das Spielfeld ist spielbar.
  7. Erlaube eine individuelle Spielfeldgröße
    1. Füge ein Input-Feld mit einer passenden Beschreibung hinzu
    2. Hier kann man eine Zahl zwischen 4 und 7 eintragen
    3. Wird das Spiel gestartet, ist die eingetragene Zahl die Anzahl der Spalten und Zeilen auf dem neuen Spielfeld
    4. Mit steigender und sinkender Zahl erhöht sich bzw. sinkt auch die Anzahl der kleinen Divs
  8. Ermögliche eine Auswahl an Gruppen
    1. Die Bilder mit den Namen sind einzelnen Gruppen zugeteilt
    2. Erstelle eigene "Obergruppen" und setze sie in sinnvolle Beziehung zu den vorhandenen Gruppen (z.B. DAKO->Entwicklung->Frontend)
    3. Überlege dir eine Darstellung, die dem Nutzer die Gruppenstruktur zeigt
    4. Der Nutzer soll die Möglichkeit haben, verschiedene Gruppen auszuwählen
    5. Wird eine Obergruppe ausgewählt, werden automatisch alle Untergruppen ausgewählt
    6. Wird das Spiel gestartet, werden nur Bilder aus den ausgewählten Gruppen zum Aufbau des Spielfelds benutzt
